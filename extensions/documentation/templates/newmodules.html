How to implement a new module


In this section we will present you how to setup a new module.

The DataDiVR uses modules to encapsle related tools together. Therfore, the DataDiVR has a some usefull custom HTML elements to allow easy implementation of new modules which are minimizable as well as maximizable to have the best possible overview about your tools.

There are more custom HTML elements
to discover.

Module Container: mc-module


The mc-module tag works exactly as a div but gives the user the freedom to write only the genuinely necessary UI code. By using this tag to create a new HTML element there are no further configurations necessary except to set a display name using the custom "name" attribute as well as to give the element a unique id to track its state in the backend. Following, there is an on how to setup a module correctly, however feel free to checkout templates/mAnnotation.html as an examplary implementation. 

				
<mc-module id="module-annotation" name="Annotation Module">
	<div id="your content"
		This also is your content...
	</div>
</mc-module>

Alternative Dropdown: mc-dropdown-no-header



Sometime you might need only a dropdown as a main selection tool in your module. To keep the advantages of the mc-dropdown template without the display name there is the mc-dropdown-no-header HTML element. It works exactly the same as mc-dropdon.

To check out how its working have a look at templates/mAnalytics.html in the backend to see how it's working.
