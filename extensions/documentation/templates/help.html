DataDiVR Web App Documentation

The DataDiVR Web App is a flask webserver that acts as a game server ( mostly referred to as "backend" - the python part) and as a frontend (all the html and js files) the user interfaces for the VR Module. It also has functionality that is meant to be used from a normal webbrowser (uploading files, preparing data) that can be used in conjunction with a vr session in a cross-platform approach
There is also the WebGL preview, that allows the user to view and interact with networks without VR hardware directly in the browser.
Note that this works only up to a certain size, whereas the Unreal Engine VRModule can handle much bigger networks.

Backend
- one per session
- serves the html pages that act as user interfaces (UI) in VR
- multicast changes to the UI to all connected clients
- serves networkdata as textures
- performs analytic tasks on network data
- holds the state of the session

Frontend
- client side (can run on a different machine than server)
- one for each player
- notify server of useractions (socketIO)
- communicates with VRModul via WebUI.js ("Middleman")

VRModule
- binary, can't be changed
- has fixed api
- in game webbrowser (Chromium)
- WebUI plugin for bi directional communication with Frontend
- downloads big textures directly from server
- config file tells it which server to connect to and which html pages to load
